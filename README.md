# Гонки дронов

Данный репозиторий содержит файлы для подсчета количества рамок, через которые проетает дрон. Камеры должны находится на самом дроне, что может сэкономить немного денежных ресурсов, так как не нужно будет добавлять по всей трассе дополнительные детекторы/камеры и др.

## Минимально необходимые файлы

Для простого тестирования работы данной программы нужно всего 4 файла: last.pt, process_vid.py, utils.py и видео-файл с дрона (vid.mp4). 

Пример запуска:
```
python3 process_vid.py
```

После этого запустится процесс обработки видео. После его завершения в той же папке будет файл vid_other.mp4, в котором будет обработанное видео. Пример обработанного видео есть по ссылке: [VK](https://vk.com/video291734754_456239072?list=ln-YLcK6o1SZGFrLbLKzV)

## Конфигурация

### Нейросеть

В качестве основного элемента используется нейросеть YOLOv11s. Она определяет местоположение и размеры зелёных и красных рамок на экране. Все изображения и надписи для обучения есть в папках images и objects. Если _рамки другие/другого цвета/хочется больше картинок для обучения_, используйте файл create_images.py

Он автоматически откроет файл vid.mp4, после чего вы сможете нарезать каждый кадр, выделяя нужные вам элементы. К сожалению, все элементы, которые вы нарежете таким образом, будут записываться как элементы 0 типа. Но можно достаточно легко добавить возможность выделения элементов по типам. Вот только, думаю, в таком случае вам лучше использовать [Label Studio](https://labelstud.io/).

Описание управления будет в самом [конце](#управление).

### Константы

В начале файла utils.py есть константы, которые можно менять по своему усмотрению. Ниже описание каждого:

* `RACE_INFO = None` - Используется для хранения информации о предыдущих рамках. Не менять.

* `RACE_RESULTS = None` - Используется для хранения текущих результатов гонки. Является списком.

* `RACE_COUNT = 4` - Количество участников в забеге. Влияет на размер предыдущих двух списков. Номер участника передается вторым параметром у функции.

* `MIN_CURRENCY = 0.8` - Минимальная уверенность при определении рамки у нейросети.

* `FRAMES_COUNT = 2` - Минимальное количество кадров, на которых должна быть определена рамка.

* `FRAMES_DELAY = 3` - Количество кадров "запоминания". По-другому, максимальное количество кадров между определением кадров с рамками + количество кадров для подсчёта, после пролёта через рамку.

* `TOTAL_SIZE = 0.4` - Минимальный итоговый размер рамки, который должен быть во время пролёта через неё. Коэффициент, показывающий отношение площади покрытия обнаруженной рамки к полной площади камеры.

* `SIZE_RANGE = 0.1` - Возможное изменение уменьшение размера рамки. Используется, чтобы случай подлёта к рамке и отлёта от неё не засчитывался.

* `MODEL = YOLO("last.pt")` - Модель для обнаружения рамок. Путь к файлу может быть передан 1 аргументом при запуске файлов.

## Как работает

Работает подсчёт достаточно просто: результаты гонки хранятся в глобальных переменных и обновляются по мере вызова функции process_frame. Она засекает все рамки, найденные нейросетью, отсекает все, у которых уверенность меньше `MIN_CURRENCY` и берёт из оставшихся рамку с максимальной площадью покрытия. Далее, на основе предыдущих данных о рамках, либо обновляет эти данные, либо игнорирует её.

Информация о предыдущих рамках хранится такая: класс, оставшееся количество кадров «запоминания», размер предыдущей рамки и количество кадров, в которых обнаружилась рамка.

В случае, если кадры «запоминания» прошли, то прошлые данные о рамках стираются и заменяются текущими. Счетчик количества кадров с рамкой уменьшается до 1. Количество кадров «запоминания» изменяется на `FRAMES_DELAY`.

Если же текущий размер рамки больше либо чуть меньше (контролируется параметром `SIZE_RANGE`)  предыдущего размера, то данные о размере обновляются, количество кадров «запоминания» изменяется на `FRAMES_DELAY`, а количество кадров с рамкой увеличивается на 1.

Если же текущий кадр — последний для «запоминания», то в случае, если размер последней рамки больше или равен `TOTAL_SIZE`, а количество кадров с обнаруженной рамкой больше или равно `FRAMES_COUNT`, делается вывод о том, что данная рамка пройдена успешно.

Далее, в зависимости от того, была пройдена зелёная либо красная рамка счётчик пройденных рамок либо увеличивается, либо обнуляется.

## Описание функций и запуска файлов

Основной файл с функциями - utils.py

* `process_frame(frm, ind = 0)` - Основная и единственная функция для подсчёта. В качестве 1 параметра передаётся кадр, в котором будут выделены все рамки и в левом верхнем углу будет счёт текущего участника. В качестве 2 параметра - индекс участника (с 0 до `RACE_COUNT - 1`)

* `show_frame(frm)` - Простейшая функция для показа текущего кадра. Space/enter - следущий кадр. q/esc - выйти.

При запуске файла process_vid.py можно указать до 3 дополнительных аргументов:
```
python3 process_vid.py [last.pt (файл с нейросетью)] [vid.mp4 (файл с видео для обработки)] [vid_other.mp4 (имя файла с видео для записи)]
```
Данные аргументы должны идти именно таком порядке и именно на таком месте. То есть, если вы ходтите изменить только имя файла для вывода, то нужно будет также указать и имена файлов с нейросетью и начальным видео.

## Управление в create_images.py

* Стрелочки - переместить текущую область для выбора
* 1-9 - сохранить текущую область в соответствующий контейнер (порядок не важен, можно сохранить 2 элемента под номерами, к примеру, 1 и 9). Для удаления области, нажать соответствующую кнопку 2 раза.
* w/e - уменьшить/увеличить размер текущей области в ширину
* s/d - уменьшить/увеличить размер текущей области в высоту
* q/esc - выйти
* enter - сохранить и перейти на следующий кадр
* space - перейти на следующий кадр без сохранения
* -/+ - уменьшить/увеличить шаг перемещения и изменения размера. Текущий шаг показывается слева сверху
